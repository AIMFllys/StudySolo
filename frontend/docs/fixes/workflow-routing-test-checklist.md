# 工作流路由修复 - 测试清单

## 修复概述
修复了新建工作流时 `/workspace/new` 路由被当作工作流ID导致的 HTTP 500 错误。

## 测试环境准备
- [ ] 前端服务运行在 `http://localhost:2037`
- [ ] 后端服务运行在 `http://localhost:2038`
- [ ] 已登录用户账号
- [ ] 用户有剩余工作流配额
