# 2026-03-05 更新日志

## 【面向用户】

### v0.3.3 滑动拼图验证码体验升级
1. **极致视觉与体验升级**：登录、找回密码等安全验证环节全面抛弃传统简陋滑块，升级为极具科技感、带抽象艺术背景的高清“滑动拼图”验证码。
2. **更高宽容度与精准性**：支持细腻的容错（±6px内即刻吸附），配合动态光影反馈，验证过程丝滑流畅。
3. **网络异常精准提示**：修复了网络不佳或服务维护时，界面依然“谎称”「拼合不准确，请重试」的严重误导情况。现在若请求失败会直接提示验证服务异常。

## 【面向开发者】

### v0.3.3 Puzzle CAPTCHA 架构重构与代理链路根治
1. **Canvas 机制重构与等比例映射**：彻底丢弃引发错乱的由于 CSS `w-full` 引发的弹性缩放。采用了同像素级 `width=320` 双层轨独立渲染画布 (`canvas`与`block`) + `getImageData` 的专业实践，实现了真正的像素级绝对匹配。
2. **精准 Jigsaw 几何修正**：重新修复了旧代码中 `drawPiecePath` 下的弧度切面错误（不再会生成半成品的圆形），目前具备高度拟真度的缺口内凹与外凸。 
3. **Next.js IPv6 localhost 代理坑点修复**：深度排除了因为 Node.js 17+ 采用 IPv6 `::1` DNS 默认解析导致由于 FastAPI (0.0.0.0 IPv4) 无法接收转发请求引发全局 500 假死的问题。`next.config.ts` 中的 fallback 从 `localhost` 全部修正为 `127.0.0.1`。
4. **服务端 Hydration 黑区与字体加载解耦**：清还技术债，通过 `useMounted` 等待屏障隔绝客户端 Canvas 随机生成和 Server 的 SSR DOM Diff 冲突；利用标准 `<link>` 取代违规的 `<Script>` 抓取 Material Symbols，净化了终端错误日志。
5. **FastAPI python-multipart 支持**：解决启动 Python 后端时因为遗落 `python-multipart` 在 `requirements.txt` 导致的 FormData 处理崩溃。
