### 【面向开发者】

#### v0.1.007 ⚙️ 工作流执行引擎 + SSE 流式推送 + 前端渲染链路
- **拓扑排序执行引擎**（`app/services/workflow_engine.py`）：根据 edges 对节点进行拓扑排序，按依赖顺序逐节点调用对应 AI 模型。每个节点执行前自动将 Implicit_Context（暗线上下文，包含主题、语言风格、核心大纲等共享信息）注入该节点的 System Prompt。节点执行失败时立即停止，所有下游节点保持 pending 状态不再执行。
- **SSE 流式推送端点**（`GET /api/workflow/{id}/execute`）：基于 `sse-starlette` 实现 Server-Sent Events，向前端推送 4 种事件——`node_status`（节点状态变更，如 pending→running→done）、`node_token`（AI 生成的增量文本 token）、`node_done`（单个节点执行完成，附最终输出）、`workflow_done`（整个工作流执行结束）。
- **自然语言输入组件**（`WorkflowPromptInput.tsx`）：文本输入框 + "生成工作流"按钮，调用 `POST /api/ai/generate-workflow`，返回的 nodes/edges 注入 Zustand Store 后画布自动渲染出节点和连线。
- **Markdown 渲染组件**（`NodeMarkdownOutput.tsx`）：集成 `react-markdown` + `remark-gfm`（GitHub 风格表格/任务列表）+ `remark-math` + `rehype-katex`（数学公式）+ `rehype-raw`（原始 HTML），代码块使用 `shiki` 语法高亮。流式输出模式下使用 `streamdown` 增量解析器直接操作 DOM，避免 react-markdown 全量重渲染导致的闪烁。
- **AI Step 节点组件**（`AIStepNode.tsx`）：画布上的自定义节点卡片，宽度 320px，顶部显示标题和状态图标（pending ⏳、running 🔄带呼吸灯动画、done ✅、error ❌），中部集成 NodeMarkdownOutput 渲染 AI 输出（最大高度 200px 可滚动），运行中显示闪烁光标动画，完成后右上角显示复制按钮。移动端点击节点弹出底部抽屉查看内容。
- **SSE 接收 Hook**（`use-workflow-execution.ts`）：前端通过 EventSource 接收后端 SSE 事件流，根据事件类型实时更新 Zustand Store 中对应节点的 status 和 output。`node_token` 事件增量追加到节点输出内容，`node_done` 设置最终输出和状态。所有节点完成后自动关闭连接，连接断开时自动重连最多 3 次。
- **属性测试**：拓扑排序执行顺序（Property 15）、暗线上下文生命周期（Property 14）、错误传播停止执行（Property 18）、SSE 事件类型合规性（Property 16）、SSE 事件驱动 Store 更新（Property 17）。

#### v0.1.008 🌐 MVP 端到端集成 + 生产部署基建
- **Landing 首页**（`app/page.tsx`）：产品介绍页，包含品牌标语、功能特性展示和"开始使用"CTA 入口按钮。
- **错误与加载状态处理**：网络失败→重试提示、AI 超时→超时信息、Token 不足→配额提示。工作流列表和画布数据加载中显示 Loading 骨架屏占位动画。
- **安全加固**：确认 CORS 中间件仅允许指定域名访问、所有 AI API Key 仅存储在后端 `.env` 中前端零泄露。
- **部署脚本**：`scripts/deploy-frontend.sh`（拉取代码 → `pnpm build` → PM2 重启，端口 2037）和 `scripts/deploy-backend.sh`（拉取代码 → `pip install` → Gunicorn 重启 workers，端口 2038）。
- **Nginx 反向代理**（`scripts/nginx.conf`）：`/` 代理到前端 2037、`/api/` 代理到后端 2038；SSE 长连接支持（`proxy_buffering off`、`proxy_read_timeout 300s`）；静态资源 365 天 immutable 缓存；安全响应头（X-Frame-Options、X-Content-Type-Options、X-XSS-Protection、Referrer-Policy）；HTTPS 强制跳转（Let's Encrypt SSL）。
- **Supabase 保活**（`.github/workflows/supabase-keepalive.yml`）：GitHub Actions 每 5 天自动调用 Supabase 健康检查 API，防止免费实例因不活跃被暂停。
- **Swap 配置**：部署脚本中包含 2GB Swap 分区配置命令，防止小内存 ECS 服务器 OOM。
- **属性测试**：CORS 来源限制（Property 20）、Pydantic 请求参数校验（Property 21）。
