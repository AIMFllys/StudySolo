#### v0.1.004 🔐 前后端认证体系完整实现
- **后端认证 API**（`app/api/auth.py`）：实现注册（Supabase Auth 创建用户 + 邮箱验证触发）、登录（凭据验证 → access_token + refresh_token 写入 HttpOnly Secure SameSite=Lax Cookie）、登出（清除 Cookie + 会话失效）、Token 无感刷新、获取当前用户信息共 5 个端点。
- **JWT 中间件**（`app/middleware/auth.py`）：对受保护路由进行 Bearer Token 校验，无效或过期返回 401。
- **CORS 中间件**（`app/middleware/security.py`）：仅允许 `CORS_ORIGIN` 指定域名跨域访问。
- **前端 Supabase SSR**：创建浏览器端（`client.ts`）和服务端（`server.ts`）两套 Supabase 客户端，以及 `updateSession` 会话刷新辅助函数。
- **Next.js 路由守卫**（`middleware.ts`）：保护 `/workspace`、`/settings`、`/history` 路由，未认证重定向到 `/login?next={原始路径}`。
- **登录/注册页面**：邮箱+密码表单，桌面端居中卡片式布局，移动端全屏适配。
- **跨 Tab 登出同步**：通过 `supabase.auth.onAuthStateChange` 监听登出事件，同步所有标签页。
- **属性测试**：Cookie 安全属性验证（Property 1）、JWT 无效令牌拒绝（Property 2）、路由守卫重定向（Property 3）。
