### 【面向开发者】

#### v0.2.001 🏗️ 插件化节点架构重构

**概述**：将 AI 工作流节点系统从单文件混合架构重构为模块化插件架构。这是一次纯后端+前端渲染层的架构升级，不改变用户交互流程和 API 契约。

**新增后端模块（26 个文件）**：

- **节点基础框架** (`nodes/`)
  - `_base.py`：`BaseNode` 抽象基类，通过 `__init_subclass__` 实现定义即注册（零手工注册代码）。标准化 `NodeInput` / `NodeOutput` Pydantic 模型。`system_prompt` 属性自动从同目录 `prompt.md` 加载
  - `_mixins.py`：`LLMStreamMixin`（标准流式 LLM 调用封装）+ `JsonOutputMixin`（三级 JSON 修复策略：直接解析 → 去代码围栏 → 提取首尾括号）
  - `_categories.py`：节点分类枚举（input / analysis / generation / interaction / output）
  - `__init__.py`：`pkgutil.walk_packages` 自动扫描所有 `node` 子模块，触发注册并导出 `NODE_REGISTRY`

- **9 个节点插件**（每个独立文件夹：`__init__.py` + `node.py` + `prompt.md`）
  - `input/trigger_input`：非 LLM，透传用户输入
  - `analysis/ai_analyzer`：JSON 输出 + 后处理校验
  - `analysis/ai_planner`：工作流规划 JSON 输出
  - `generation/outline_gen`：Markdown 大纲
  - `generation/content_extract`：知识提炼
  - `generation/summary`：学习总结
  - `generation/flashcard`：**修复 JSON/Markdown 通道混淆 bug**，现使用 `JsonOutputMixin` 强校验
  - `interaction/chat_response`：对话回复
  - `output/write_db`：非 LLM，数据库持久化（TODO 真实写入逻辑）

- **新执行引擎** (`engine/`)
  - `executor.py`：全新编排引擎（~100 行 vs 旧 229 行）。从 `NODE_REGISTRY` 动态查找节点类（消除 if/else 链），调用 `node.post_process()` 做输出后处理。**关键改进**：通过 `context.py` 的 `build_upstream_map` 实现基于 Edge 的精确上下文传递——每个节点只接收直接上游输出，不再全量拼接
  - `context.py`：上游映射 + 下游映射 + 错误传播（`get_all_downstream` 实现级联跳过）
  - `sse.py`：SSE 事件格式化工具

- **通用工具** (`utils/`)
  - `token_counter.py`：用 `tiktoken` BPE 编码替代 `split()` 的中文 Token 计数 bug
  - `output_truncator.py`：按句子边界智能截断，防止下游节点上下文溢出

- **节点清单 API** (`api/nodes.py`)
  - `GET /api/nodes/manifest`：聚合所有已注册节点的元数据（type, category, description, icon, color, output_format），供前端动态生成节点面板

- **开发者文档** (`nodes/CONTRIBUTING.md`)
  - 完整的新增节点操作指南：7 步流程 + 3 种模板（标准 LLM / JSON 输出 / 非 LLM）+ 改动 Checklist + 数据流全链路图 + FAQ

**后端修改（3 处）**：
- `api/workflow.py` L15：`execute_workflow` 导入从 `services.workflow_engine` 切换到 `engine.executor`
- `api/router.py`：新增 `nodes_router` 导入和 `/api/nodes/` 路由注册
- 旧文件 `services/workflow_engine.py`：保留不删除，新引擎完全替代其功能

**新增前端模块（7 个文件）**：

- **渲染器注册表** (`nodes/index.ts`)：前端等价物 `RENDERER_REGISTRY`，`getRenderer(nodeType)` 动态匹配渲染器。新增类型无需改 `AIStepNode.tsx`

- **5 个专属渲染器** (`renderers/`)
  - `MarkdownRenderer.tsx`：默认渲染器，复用 `NodeMarkdownOutput`
  - `FlashcardRenderer.tsx`：JSON 解析 → 可翻转交互卡片（渐变背景 + 翻转动画）
  - `OutlineRenderer.tsx`：大纲渲染（预留可折叠树状 UI 扩展点）
  - `JsonRenderer.tsx`：格式化 JSON 展示（暗色代码主题）
  - `PassthroughRenderer.tsx`：非 LLM 节点紧凑状态指示器

**前端修改（1 处）**：
- `AIStepNode.tsx`：移除硬编码 `NodeMarkdownOutput`，改为 `const Renderer = getRenderer(nodeType)` 动态委托
