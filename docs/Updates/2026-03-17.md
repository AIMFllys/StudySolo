# StudySolo 2026-03-17 产品更新履历

## 【面向用户】

### v0.3.5 智能工作流体验与云端同步引擎全面焕新
我们非常荣幸地宣布，StudySolo 迎来了工作流引擎底层架构与交互体验的重要迭代。本次更新着力于为您提供无缝、丝滑且绝对可靠的云端创作环境。

1. **金融级可靠性的云端实时同步引擎**：
   - 数据安全是灵感创作的生命线。我们彻底重构了工作流的同步机制，修复了在极端高频编辑场景下本地数据偶发未同步至云端的问题。现在，您的每一次连线、每一个属性调整都将被高精度捕捉并自动落盘。即使遭遇意外中断，重新进入工作区亦可百分之百无损无缝恢复。
2. **渐进式「骨架屏」与手绘美学加载序列**：
   - 告别单调的白屏等待。在编排画布和节点初次挂载时，系统引入了与平台主视觉深度契合的“纸质感知”骨架屏（Skeleton UI）以及动态的“手绘电路板”加载动画。从数据传导到视觉呈现，过程极为丝软顺畅，大幅消解了等待过程中的突兀感。
3. **AI 智能生成状态感知窗**：
   - 增强了与 AI 协同构建工作流时的透明度。当您下拨指令让 AI 助手构思节点结构时，画布中央将呈现专属的“生成占位节点”动效。您可以直观地感知 AI 思维链的运转进程，时刻掌握工作流构建的生命周期。

## 【面向开发者】

### v0.3.5 工作流同步架构重构 (Periodic Snapshot & Local First)
根据 Github 提交记录 `feat:优化工作流云端同步功能，新增懒加载` 及系统迭代基线，本次改版深度优化了 React 生命周期与全局状态持久化之间的性能瓶颈：

1. **同步时钟重置：从 Debounce 跃升至 Interval Hash Snapshot**：
   - 废弃了依赖 `isDirty` 副作用触发的传统防抖限流同步模型。系统现转为每 2 秒利用高精度 Hash 值轮询比对本地节点拓扑差异，借由脱离 React Lifecycle 的 `setInterval` 独立线程，彻底根除高并发修改导致的 `Update 500` 或队列截断遗漏更新问题。
2. **前后端执行锁 (Execution Lock) 与竞态控制**：
   - 针对“运行态”和“编辑态”交叠产生的云端脏数据覆写问题，引入了刚性执行锁策略。当接收到后端引擎派发的 SSE (Server-Sent Events) 执行结果时，前端 `useWorkflowSync` 将自旋暂停定时快照上传，严格防御后端执行流状态被前端的历史快照强行覆盖。
3. **离线首选：IndexedDB 兜底容灾策略**：
   - 同步链路全面切入“Local First”架构，写链路更为 `LocalForage (IndexedDB) -> Web Worker 缓冲限流 (8s) -> Supabase Cloud`。断网或浏览器崩溃时，本地缓存亦能在恢复后识别标记为“脏快照（Dirty Snapshot）”，优先驱动补偿推流。
4. **GeneratingNode 状态机隔离与懒加载化**：
   - 独立解耦承载 AI 编排中间态的图节点实体，优化了 React Flow 在批量灌入节点簇时的首帧渲染运算强度。懒加载架构分离打包提高了工作流初次调入性能。
