# StudySolo 2026-03-26 产品更新履历

## 【面向用户】

### v0.7.0 商业级计费中枢、协作引擎与画布深度融合架构 (Phase Completion)
本次迭代代表了 StudySolo 协作能力演进中的关键转折。我们全面完备了涵盖复杂控制流与精细化资源账款管控的交互框架，将企业级生产效能注入到每一块节点空间中。

1. **核心成本洞察：CNY 全生命周期 AI 账单与聚合大盘**：
   - 为确保您能清晰掌控资产流动与投资回报，个人中心引入了首版精细计费聚合仪表盘（Billing Dashboard）。所有支出统计逻辑正式且全面跃迁至标准的人民币（CNY）法币口径映射。您可以通过图表纵览模型供应商的阶梯支出情况，每一分消耗、每一笔投资，皆可管可溯。
   - **智能模型商品目录网关 (Model Catalog API)**：所有节点内置模型选择模块现已无缝挂载云端目录实时渲染架构，彻底破除本地固化名单的短板！即刻为您展示各路模型最新的平台官方定价、性能等级对齐依据，并辅助引导特权节点解锁路径。
2. **空间多维连通：多层级工作流全局协作网络 (Collaborative Marketplace)**：
   - 社区分享绝非仅供观瞻。主展示面板正式完成了极高安全级别的前后端渲染隔离架构，提供无损沉浸化分享页（URL Path Separation `/s/:id` & 编辑向 `/c/:id`），最大程度防御跨域编辑污染。
   - **在线群聚态 (Multiplayer Presence)**：您已可以通过管理面板对合作者分配详细诸如只读（Viewer）或最高写权（Editor）的精确实权调配证书；协作进行中，画布苍穹区更将为您悉数点亮所有即刻在线的高频协作人员序列信标。
3. **“零黑盒”全知节点体系：三级插槽式洞察与控制流全景化**：
   - 彻底废除了繁杂无序的交叉连线乱象。所有的执行连接规则皆通过标准的“唯一序列线”传输。我们在画布交互层面为您植入了分支预告标签及延迟执行的读秒控制刻度盘（⏱），双击即可快捷变调，让流转指令彻底可见。
   - **专属聚合循环沙盒 (Secure Loop Container)**：突破单向进程局限，全新发版具备绝对嵌套纵深包围感及隔离层级的循环容器节点群（Loop Group）。将关联项落入即可激活捕获及安全锁定，结构层次分明的同时更可为您穿透呈现实流信息（`第 X 循环回合/总局 N 遍透视`）！全景动态投射一目了然！

## 【面向开发者】

### v0.7.0 RLS 细粒度资源管控、双层目录网关与画布核心事件解耦
26 日是高并发数据重组与架构落地整合的决胜枢纽日。系统在单日完成横向覆盖近 30 余组精研代码集成节点（基于代码库追溯记录如：`consolidate workflow canvas control flow`, `Phase 1 - preview-first routing`, `complete branch panel & smoke tests` 等），于全栈维度下实现了如下关键硬核更替：

1. **AI 交易层与 Usage Ledger 全案立柱 (Models Catalog Framework)**：
   - **资源族与 SKU 分离解耦 (Dual-Layer Abstraction)**：彻底重定义底层账务记录及追踪实体，并于 DB 部署剥离原耦合参数的高维抽离架构 `ai_model_families`（抽象大基座分类网络）与 `ai_model_skus`（真实可精准计费的端侧网络）双物理映射图。确保无论外挂上游如何实施模型渠道的更迭替补，底层结算计账依旧能够将 Token 并按千位级精确铆钉于 CNY 账户下，不被任何参数偏移带走。
   - **Admin 全域监管对接**：后端总监控图强无缝直连接线前端的数据图库端，实现了（Unified Sources of Truth）数据管理端向呈现监控端的实时刷新推流同步映射，让主控中心尽悉任何流量与消费热点的转移爆发走势。
2. **React Flow 拓扑约束重构与 Edge Connection 防错总收口**：
   - **状态机对齐升维 (SSE Lifecycle Harmonization)**：梳理且全量捕获分析后端执行进程的生命链路指令发送流，前端状态树针对诸如被悬浮跳过、入队阻断休眠、循环遍历流向等特定时序(`waiting, skipped, loop_iteration`)进行了同步注册消费扩容。一举抹平客户端与服务器处理延迟不同步的断层闪烁瑕疵。
   - **嵌套级联渲染坐标轴分离算法 (loop-group-drop Algorithm)**：摒弃了 React 繁杂的深包引用以及 `useRef` 黑盒穿行捕获策略。另起基建开发 `use-loop-group-drop` 非侵占式处理层，深度且无锁捕获操作区坐标包围盒并实施子父集相对绝对算法坐标归真变换并映射到图计算引擎所标配的原生事件（`extent: parent`）中，极大降低渲染死锁隐患几率。
3. **行级数据护卫队 (Row Level Security) 鉴权拦截矩阵**：
   - **核心物理驱动门禁切割**：在框架路由端物理硬性分离外部窥查 ` /s/` 和协作深度调用 ` /c/`；部署底层 API 触发拦截 Middleware ，拒止并静默吸收无凭证游客的所有越权调用操作请求和高耗资源提取盗刷攻击。
   - 彻底引入了全数据链路协同共享的控制白名单检查阵列网络和三级精控节点筛查。保障操作人于授权合规下的稳健请求，大幅消灭后端直接处理无授权数据并导致 500 的空跑操作负荷。
4. **前端组装与编译性能极致清洗**：
   - 发起全工程 `requiresModel` 和 `User-keyed catalog cache` 元数据分离梳理清扫。针对由于服务端直出水合差异 `SSR hydration mismatch` 以及冗余组件如 `idling slip` 所遗存在界面上过度深暗虚浮阴影引发的堆栈堆积报警进行了精确抹平修正软化。
