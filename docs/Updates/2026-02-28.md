#### v0.1.011 🧩 10 处集成缺陷修复 (integration-fixes Phase 2-4)
修复前期各组件之间未正确连接的 10 处集成缺陷：
- **Nginx 域名修正**（缺陷 1）：`scripts/nginx.conf` 中遗留的 `your-domain.com` 占位符全部替换为实际域名 `studyflow.1037solo.com`，涉及 `server_name`（2 处）和 SSL 证书路径（2 处）。
- **Sidebar 工作流列表数据贯通**（缺陷 2）：`DashboardLayout`（Server Component）通过 Supabase Server Client 查询当前用户工作流元数据（id, name, updated_at），按 `updated_at` 降序排列后传递给 Sidebar 组件——此前 Sidebar 渲染的是空列表。
- **新建工作流功能接通**（缺陷 6）：新建 `DashboardShell.tsx`（Client Component）封装 Navbar 和 MobileNav，实现 `handleNewWorkflow`（调用 `POST /api/workflow/` 创建 → `router.push` 跳转到新画布页），解决 Server Component 无法绑定点击事件回调的问题。
- **代码高亮双主题**（缺陷 3）：为 NodeMarkdownOutput 新建 ShikiCodeBlock 子组件，使用 `shiki/bundle/web` 的 `codeToHtml` 替代原始 `<pre>` 标签，支持 github-light / github-dark 双主题，加载失败时优雅回退到纯文本 `<pre>`。
- **流式输出防闪烁**（缺陷 4）：streaming 模式下从 react-markdown 全量重渲染切换为 `streamdown` 增量 DOM 更新（直接操作 DOM 节点而非 React 虚拟 DOM），消除流式输出过程中 Markdown 列表和代码块的闪烁抖动。streaming 结束后切回 react-markdown + shiki 全量渲染以保证最终排版准确。自动滚动到底部。
- **RightPanel 右侧统计面板**（缺陷 5）：从 Zustand Store 读取全部 nodes，按 status 分组统计各状态节点数量，展示执行概览卡片 + 用户设置入口。桌面端 `md:flex` 显示，移动端隐藏。
- **BottomDrawer 移动端底部抽屉**（缺陷 7）：移动端点击画布节点弹出底部抽屉，展示节点标题 + 状态标签 + NodeMarkdownOutput 完整渲染。支持手势下滑关闭和遮罩层点击关闭，streaming 状态下内容实时更新。
- **RunButton 运行按钮**（缺陷 8）：工作流运行中显示"⏹ 停止"，空闲时显示"▶ 运行全部"，画布无节点时按钮禁用灰显。
- **组件集成到页面**：RightPanel 挂载到 DashboardLayout，BottomDrawer 挂载到 WorkflowCanvas（移动端 onNodeClick 触发），RunButton + WorkflowPromptInput 挂载到 `workspace/[id]/page.tsx`。
- **属性测试**：Sidebar 渲染一致性（Property 1）、Nginx 域名配置（Property 10）、Shiki 输出有效性（Property 2）、streaming 内容等价性（Property 3）、BottomDrawer 完整性（Property 4）、RightPanel 统计准确性（Property 5）、PromptInput 数据一致性（Property 6）、RunButton 状态映射（Property 7）。

#### v0.1.012 💾 工作流运行结果自动保存 (integration-fixes Phase 5)
- **save_callback 机制**：为 `execute_workflow` 新增 `save_callback` 可选参数。工作流全部节点执行完成后、发送 `workflow_done` 事件前，调用 callback 将带有最新 output 的 `updated_nodes` 写回 Supabase workflows 表。如果保存失败，向前端发送 `save_error` SSE 事件告知（不中断整体流程，输出内容仍然显示在前端），用户可以手动复制内容。
- **workflow API 集成**：在 `workflow.py` 的 `execute_workflow_sse` 中实现 `_save_results` 回调函数，传入 execute_workflow 调用。
- **属性测试**：自动保存完整性（Property 8，验证 callback 接收到的 nodes 与执行后 Store 中的一致）、save_error 容错（Property 9，验证保存失败时 SSE 仍然包含 save_error + workflow_done 两个事件）。

- **Admin Shell 布局**（`admin-analysis/layout.tsx`）：Client Component，包含 AdminSidebar（左侧导航）+ AdminTopbar（顶部栏）+ children 内容区。

- **AdminTopbar**（`components/business/admin/AdminTopbar.tsx`）：显示当前管理员用户名 + 登出按钮。
- **Admin 登录页**（`admin-analysis/login/page.tsx`）：Glass Morphism 卡片样式登录表单，支持错误提示显示（如"用户名或密码错误"）、账号锁定时显示剩余等待倒计时、首次登录（`force_change_password=true`）时自动弹出改密表单。
