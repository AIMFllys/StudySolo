# StudySolo 项目架构全景

> 最后更新：2026-03-26
> 文档编码：UTF-8（无 BOM） / LF
> 事实源：`frontend/package.json`、`backend/requirements.txt`、`backend/app/api/*`、`frontend/src/types/*`、`backend/config.yaml`、`supabase/migrations/*`、`.gitmodules`

## 1. 项目定位

StudySolo 是 1037Solo 生态中的 AI 学习工作流平台。

核心流程：

1. 用户在前端画布组织学习工作流
2. 后端按 DAG 顺序执行节点
3. AI 输出通过 SSE 持续回流
4. 结果进入知识库、导出、持久化与统计账本

## 4. 前端架构

### 4.1 分层

```text
frontend/src/
├─ app/                      # App Router 路由层
├─ features/                 # 领域层
├─ services/                 # API 访问
├─ stores/                   # Zustand 状态
├─ types/                    # TS 契约
├─ hooks/                    # 通用 hooks
└─ styles/                   # 全局样式
```

### 4.2 主要业务域

- `features/workflow/`：画布、节点、工具栏、执行、AI 侧栏
- `features/admin/`：后台仪表盘、用户、工作流、通知、评分、会员、配置、审计
- `features/knowledge/`：知识库上传、列表、查询
- `features/auth/`：登录、注册、验证码、密码重置
- `features/settings/`：个人设置

### 4.3 状态与数据流

- 工作流编辑状态由 `use-workflow-store.ts` 维护
- 服务调用通过 `services/*` 统一发起
- 工作流执行与 AI 聊天通过 SSE 流式返回
- 前端权威类型位于 `frontend/src/types/*`

## 5. 后端架构

### 5.1 分层

```text
backend/app/
├─ api/                      # 路由定义
├─ models/                   # Pydantic 契约
├─ engine/                   # DAG 执行与 SSE
├─ nodes/                    # 插件化节点系统
├─ services/                 # 业务服务
├─ prompts/                  # Prompt 模板
├─ core/                     # 配置、依赖、数据库
├─ middleware/               # JWT / admin / security
└─ utils/                    # 通用工具
```

### 5.2 关键模块

- `api/router.py`：统一挂载所有 `/api/*` 路由
- `engine/executor.py`：工作流执行引擎
- `nodes/_base.py`：节点基类
- `services/ai_router.py`：多平台模型调用与降级
- `services/ai_catalog_service.py`：AI 模型目录读取与更新
- `services/usage_ledger.py` / `usage_analytics.py`：usage 记录与统计

## 6. 工作流系统

### 6.1 节点体系

当前节点类型由 `frontend/src/types/workflow.ts` 定义：

- 输入：`trigger_input`、`knowledge_base`、`web_search`
- 分析：`ai_analyzer`、`ai_planner`、`logic_switch`、`loop_map`
- 生成：`outline_gen`、`content_extract`、`summary`、`flashcard`、`compare`、`mind_map`、`quiz_gen`、`merge_polish`
- 交互：`chat_response`
- 输出：`export_file`、`write_db`
- 结构：`loop_group`

### 6.2 边与状态

- 当前边类型仅 `sequential`
- 当前节点状态：
  - `pending`
  - `running`
  - `waiting`
  - `done`
  - `error`
  - `skipped`
  - `paused`

### 6.3 SSE 事件

当前工作流 SSE 事件由 `frontend/src/types/workflow-events.ts` 对齐：

- `node_status`
- `node_input`
- `node_token`
- `node_done`
- `loop_iteration`
- `workflow_done`
- `save_error`

## 8. AI 目录、路由与计费

### 8.1 目录权威

当前 AI 模型目录的权威表不是旧的 `ai_models`，而是：

- `ai_model_families`
- `ai_model_skus`

### 8.2 路由配置

运行时任务路由由 `backend/config.yaml` 控制：

- 提供商：`dashscope`、`deepseek`、`moonshot`、`volcengine`、`zhipu`、`qiniu`、`siliconflow`、`compshare`
- 路由策略：`native_first`、`proxy_first`、`capability_fixed`
- 任务路由覆盖工作流节点与工具能力，如 `premium_chat`、`search`、`ocr`

### 8.3 计费口径

当前正式口径：

- 选型字段：`selected_model_key`
- 目录主键：`sku_id`
- 计费字段：`input_price_cny_per_million`、`output_price_cny_per_million`、`cost_amount_cny`、`total_cost_cny`

兼容层仍存在：

- `selected_platform`
- `selected_model`
- `*_usd`

但不再作为新的设计目标。

## 10. shared 层与同步边界

### 10.1 当前仓库中的 `shared/`

- `shared/` 在当前仓库里是 Git submodule
- 事实来源：`.gitmodules`
- 用途：共享类型、跨项目数据库规范、同步说明

### 10.2 Platform 中的 StudySolo

- Platform Monorepo 中的 `StudySolo/` 是 git subtree
- 相关同步流程应写在 `shared/docs/guides/subtree-sync.md`
- 不要把 `shared` submodule 与 `StudySolo` subtree 混写

## 11. 部署与运行

- 前端默认端口：`2037`
- 后端默认端口：`2038`
- 推荐本地启动：
  - 前端：`cd frontend && pnpm dev`
  - 后端：`cd backend && uvicorn app.main:app --reload --port 2038`
  - 一键：`powershell scripts/start-studysolo.ps1`
